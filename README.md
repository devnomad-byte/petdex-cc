## What is petdex-cc?

petdex-cc brings **animated desktop pets** to your Claude Code workflow. Pick any pet from [Petdex](https://petdex.crafter.run/) (1,569+ community pets), install with one command, and your companion appears on your desktop вҖ” reading code with you, celebrating completions, cheering you through errors, and growing stronger as you code.

> **How is this different from `npx petdex install`?**
> The original `petdex` command gives you a basic pet. `petdex-cc` adds a deep **Claude Code integration layer** вҖ” hooks into your coding workflow, a level progression system, AI-generated speech, time-aware greetings, and interactive effects. Your pet isn't just decoration вҖ” it *lives* in your coding session.

### Features at a glance

- **Real-time reactions** вҖ” pet changes animation when you read, edit, run commands, complete tasks, or hit errors
- **8-level progression** вҖ” Byte вҶ’ Process вҶ’ Thread вҶ’ Module вҶ’ Kernel вҶ’ Neural вҶ’ Quantum вҶ’ Singularity, with escalating visual effects
- **AI speech bubbles** вҖ” context-aware dialogue generated by Claude (or built-in preset lines)
- **Time-aware greetings** вҖ” morning, lunch, afternoon, evening, night, and midnight messages in Chinese
- **Interactive** вҖ” drag your pet anywhere, right-click for menu, rapid-click easter egg
- **System tray** вҖ” show/hide, about, quit from the taskbar
- **State persistence** вҖ” level and event count survive restarts (HMAC-signed, tamper-resistant)
- **1,569+ pets** вҖ” browse [petdex.crafter.run](https://petdex.crafter.run/) and install any community pet

## Quick Start

### Prerequisites

- [Node.js](https://nodejs.org/) >= 18
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) installed

### Install

```bash
# 1. Browse pets вҶ’ https://petdex.crafter.run/
# 2. Find one you like, note the slug (e.g. "boba")
# 3. Install:

npx petdex-cc install boba
```

## What Your Pet Does

Every Claude Code action triggers a pet reaction:

| Claude Code event | Pet animation | Speech bubble |
|---|---|---|
| Session starts | Waving | "Let's get to work!" |
| Reading a file | Review | "Reading filename..." |
| Editing a file | Idle | "Edited filename" |
| Running a command | Running | "Running command..." |
| Searching (Glob/Grep) | Waiting | "Searching..." |
| Tool fails | Failed | "Oops, something went wrong" + AI encouragement |
| Task completes | Jumping | "Task complete!" + AI celebration |
| Claude stops | Jumping | "Task complete!" (60% AI speech) |
| Claude errors | Failed | Error message + AI comfort |
| Idle (10+ min) | Waving | Time-aware Chinese greeting |
| Level up | Current state | "Level up! Kernel!" + all effects activate |

### Pet interactions

- **Drag** вҖ” click and drag to move your pet anywhere
- **Right-click** вҖ” context menu with Show/Hide, About, Quit
- **System tray** вҖ” right-click the tray icon for menu
- **Easter egg** вҖ” rapidly click your pet for escalating reactions (2 вҶ’ 4 вҶ’ 6 вҶ’ 9 вҶ’ 12 вҶ’ 15 clicks)

When an Anthropic API key is available (auto-detected from Claude Code settings or set via `petdex-cc config`), your pet generates **context-aware speech**:

| Scene | Trigger | Example AI response |
|---|---|---|
| `task_complete` | Task finishes | "еҒҡеҫ—еҘҪпјҒз»§з»ӯеҠ жІ№пјҒ" |
| `error` | Tool fails | "еҲ«жӢ…еҝғпјҢbugsйҡҫе…Қзҡ„~" |
| `idle` | Idle 10+ minutes | "иҝҳеңЁеҗ—пјҹ" |

## CLI Reference

```bash
petdex-cc install <slug>    # Download pet, configure hooks, and launch
petdex-cc start             # Start the desktop pet
petdex-cc stop              # Gracefully stop the pet
petdex-cc list              # Browse all pets from Petdex registry
petdex-cc switch <slug>     # Switch pets at runtime (downloads if needed)
petdex-cc status            # Show pet name, level, events, running status
petdex-cc uninstall         # Remove hooks, stop pet, delete all data
petdex-cc config [options]  # Configure settings
```

### Config options

```bash
petdex-cc config --api-key <key>             # Anthropic API key for AI speech
petdex-cc config --api-base-url <url>         # API base URL
petdex-cc config --cooldown <minutes>         # AI call cooldown (default: 2)
```

> API credentials are auto-detected from `~/.claude/settings.json` вҖ” most users don't need manual config.

## Configuration Reference

### Auto-detected settings

petdex-cc reads these from `~/.claude/settings.json` automatically:

| Setting | Used for |
|---|---|
| `ANTHROPIC_AUTH_TOKEN` | API key for AI speech |
| `ANTHROPIC_BASE_URL` | API endpoint for AI speech |

### Manual config

Stored in `~/.petdex-cc/config.json`:

```bash
petdex-cc config --api-key <key> --api-base-url <url> --cooldown 2
```

### Data locations

| Path | Contents |
|---|---|
| `~/.petdex-cc/pets/<slug>/` | Downloaded spritesheet + pet.json |
| `~/.petdex-cc/data/state.json` | Level, events, pet slug (HMAC-signed) |
| `~/.petdex-cc/data/state.sig` | HMAC-SHA256 signature |
| `~/.petdex-cc/hooks/` | Bridge scripts |
| `~/.petdex-cc/config.json` | User config (API key, cooldown) |

## Hooks Reference

petdex-cc registers these hooks in `~/.claude/settings.json`:

| Hook | Matcher | What triggers it |
|---|---|---|
| `PostToolUse` | `Read\|Edit\|Write\|Bash\|Glob\|Grep` | After any tool call |
| `PostToolUseFailure` | вҖ” | After a tool call fails |
| `Stop` | вҖ” | Claude finishes a response |
| `StopFailure` | вҖ” | Claude stops with error |
| `Notification` | `idle_prompt` | User is idle |
| `SessionStart` | `startup\|resume` | Claude Code launches or resumes |
| `SessionEnd` | вҖ” | Claude Code session ends |
| `TaskCompleted` | вҖ” | A task is marked complete |
